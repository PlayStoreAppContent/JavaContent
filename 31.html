Java - Multithreading

 

 

Thread :-
A thread is a single sequential( separate ) flow of control within program. Sometimes, it is called an execution context or light weight process. A thread itself is not a program.

A thread can not run on it's own( as it is a part of a program ). Rather, it runs within a program. A program can be divided into a number of packets of code ---- each representing a thread having its own separate flow of control. 

 

 

Light weight process: A thread is considered a light weight process because it runs within the context of a program and takes advantage of the resources allocated to that program. 

Heavy weight process: In the heavy weight process, the control changes in between threads belonging to different processes. ( In light weight process, the control changes in between threads belonging to same(one) process ). 

Execution contest: A thread will have its own execution stack and program counter. The code running within the thread works only within that context. 

One of the strengths of Java is its support for multithreading. All the classes needed to write a multithreaded program are included in the default imported package java.lang through class Object, class Thread and interface Runnable. 

 

 

 

 

 

 

 

JAVA - Thread Life Cycle Methods

 

 

1. run( ) :
The run( ) method is available both in Thread class and Runnable interface. We can subclass a Thread class and override the run( ) method. We write the specific code, which we would like to do by the thread, in the run( ) method. 

run( ) method is the heart of any Thread and very important in its life cycle. It defines what the Thread should do when it is running. When a thread comes out of the run( ) method, it dies.
The following program illustrates:-

public class Demo  extends Thread   

{

 public void run( )

 {

  for( int i = 0 ; i < 10 ; i + + )

   System.out.println( i ) ; // prints 0 to 9 numbers

}

 public static void main( String args [ ] )

 {

Demo d1 = new Demo( ) ;

d1.start( ) ;

}

 }

In the above program, thread d1 starts the thread and prints 0 to 9 numbers by executing its run( ) method.

 

 

2. sleep( ) :
We can temporarily stop a thread for some time from its active( runnable ) state to a pause( blocked ) state by calling sleep( ) method on it. 

The following is the method signature of the sleep( ) method as defined in Thread class:

public static void sleep( long millis ) throws InterruptedException 

When the thread is in sleep( ) condition, the thread scheduler blocks the microprocessor time to it. That is, when sleep( ) method is called on the thread, it is not eligible for processor time, even if the processor is idle. sleep( ) method throws a checked exception - InterruptedException. 

The following program illustrates the usage of sleep( ) :

public class Demo  extends Thread   

{

 public void run( )  

 {

  for( int i = 0 ; i < 10 ; i + + )    

  {

   System.out.println( i ) ; // prints 0 to 9 numbers

try   

 {

        Thread.sleep( 1000 ) ;

 }

catch( InterruptedException e )  

{

 System.out.println( "Something happened to sleep" + e) ;

  }

}  

}

public static void main( String args [ ] )  

{

  Demo d1 = new Demo( ) ;

  d1.start( ) ;

}

}

 

In the above program, sleep( ) method is activated for every 1000 milliseconds and for loop delays its execution by 1000 milliseconds for each iteration. That is, sleep( 1000 ) makes thread inactive for 1000 milliseconds. 

 

 

3. stop( ) :
Thread's stop method stops a thread permanently. The stopped thread cannot be restarted again. 

A Thread instance has three main phases in its existence: 
1. born : creation of thread, before being started.
Thread t = new Thread( ) ; Thread t is in born state.

2. runnable : execution of code in run( ), after being started and before terminating.
t.start( ) ; when thread is started, it implicitly calls run( ) method. 

3. death : After terminating, but before being garbage collected. Creating and starting a thread instance brings a thread to life. 

A thread is in active condition until it executes its run( ) method or until stop( ) method is invoked on it. stop( ) method kills a thread and terminates it and the thread object is liable for garbage collection. 

Restarting a stopped thread is impossible. Instead, we can create a new thread to run the same code. 

The following is the method signature defined in Thread class : 

public final void stop( ) This method is deprecated, as it is inherently unsafe.

7. join( ) : join( ) method makes the thread to wait until other threads finish their task. 

Generally, threads are designed to run independently of the other. join() method involves two threads. The invocation of join causes the currently executing thread to suspend execution until target thread finished its execution. 

The following is the method signature:
public final void join( ) throws InterruptedException 

join() method is overloaded. The difference between join() and wait() is join() method does not involve in synchronization( that is, no locking of data). wait() method is used by JVM in thread synchronization. Other way, join() and wait() methods are used in thread communication. 

 

 

8. yeild( ) :
It is the thread scheduler to decide which thread should be given next processor time. This is decided by the waiting time of the thread for processor and its priority. Threads with more priority will definitely get first preference than lower priority threads. If many threads of the same priority exist, they execute in round robin fashion. 

If yield( ) method is called on a thread, it voluntarily gives off its processor time to other threads. yield( ) method yields its time to other threads of same priority only. 

When the thread gives off its time, it is the thread scheduler to decide again which thread is to be given next slice of processor time. The following is the method signature defined Thread class :

public static void yield( )

 

 

Multithreading is a process of executing multiple threads simultaneously. Thread is basically a lightweight sub process, a smallest unit of processing. Multiprocessing and multithreading, both are used to achieve multitasking.

But we use multithreading than multiprocessing because threads share a common memory area. They don't allocate separate memory area so save memory, and context-switching between the threads takes less time than processes. Multithreading is mostly used in games, animation etc. 

Threading is a facility to allow multiple activities to coexist within a single process. Most modern operating systems support threads, and the concept of threads has been around in various forms for many years.

Java is the first mainstream programming language to explicitly include threading within the language itself, rather than treating threading as a facility of the underlying operating system.
